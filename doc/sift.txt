INTRODUCTION                                               *sift-intro*

sift.nvim provides a Codex-backed agent session inside Neovim.

Core behavior:

  - session bootstrap against a git repository
  - a right-side panel-first workflow for prompting and review
  - a multi-line prompt composer inside the panel
  - a session baseline ref created under `refs/sift/`
  - a scratch panel for prompts and backend status
  - a nonblocking `codex exec --json` wrapper
  - baseline diff scanning into pending review hunks
  - inline pending hunk rendering and refresh/navigation commands
  - accept/reject actions at hunk, file, and session scope

Start a session with: >
  :SiftPanelToggle
<
Send a prompt from the panel with: >
  <CR>
<

==============================================================================
COMMANDS                                                 *sift-commands*

                                                        *:SiftStart*
:SiftStart
  Start a sift session for the current repository and create a baseline ref.

                                                        *:SiftStop*
:SiftStop
  Stop the active session for the current repository and delete its baseline
  ref.

                                                       *:SiftPrompt*
:SiftPrompt [text]
  Send a prompt to Codex. With no argument, sift.nvim focuses the panel
  prompt.

                                                  *:SiftPanelToggle*
:SiftPanelToggle
  Toggle the session panel for the current repository. By default this also
  starts a session automatically when one is not active yet.

                                                   *:SiftAcceptHunk*
:SiftAcceptHunk
  Accept the pending hunk under the cursor. The current text stays in the
  workspace and the hunk is removed from sift's pending-review state.

                                                   *:SiftRejectHunk*
:SiftRejectHunk
  Reject the pending hunk under the cursor by restoring baseline-equivalent
  text for that hunk region.

                                                   *:SiftAcceptFile*
:SiftAcceptFile
  Accept all pending hunks in the current file.

                                                   *:SiftRejectFile*
:SiftRejectFile
  Reject all pending hunks in the current file by restoring the file from the
  session baseline ref.

                                                    *:SiftAcceptAll*
:SiftAcceptAll
  Accept all pending hunks in the current session.

                                                    *:SiftRejectAll*
:SiftRejectAll
  Reject all pending files in the current session by restoring them from the
  session baseline ref.

                                                     *:SiftNextHunk*
:SiftNextHunk
  Jump to the next pending hunk in the active sift session.

                                                     *:SiftPrevHunk*
:SiftPrevHunk
  Jump to the previous pending hunk in the active sift session.

                                                      *:SiftRefresh*
:SiftRefresh
  Recompute pending hunks against the session baseline ref.

==============================================================================
CONFIGURATION                                              *sift-config*

Example: >
  lua require('sift').setup({
    codex = {
      sandbox = 'workspace-write',
    },
  })
<

Default options: >
  {
    codex = {
      bin = 'codex',
      sandbox = 'workspace-write',
      model = nil,
      profile = nil,
      extra_args = {},
      config_overrides = {
        model_reasoning_effort = 'medium',
      },
    },
    panel = {
      auto_start = true,
      height = 12,
      position = 'right',
      width = 50,
    },
    logging = {
      notify = false,
      level = vim.log.levels.INFO,
    },
  }
<

Workflow: >
  :SiftPanelToggle
  type a prompt in the panel
  press <CR>
  :SiftNextHunk
  :SiftAcceptHunk
<

Notes: >
  The panel opens in a right-side split by default.
  Use <S-CR> in the panel to add a new prompt line.
  <C-j> is kept as a fallback when the terminal does not send Shift+Enter
  distinctly to Neovim.
  Use @path/to/file.lua in the panel prompt to reference tracked files.
  Press <Tab> in the panel prompt to complete @file references.
  Use za on a references line to expand or collapse the payload sent to Codex.
  Use <CR> or gf on a pending file, pending hunk, or referenced file line to
  jump to it from the panel.
  The panel includes a pending-review section so review can stay split-pane,
  with the panel on the right and source files on the left.
  Use ]s and [s in the panel to jump between pending hunks.
  Use gr to refresh, gA to accept all, and gR to reject all.
  The panel shows a live activity line while Codex is working and while sift
  is refreshing review state.
  gitsigns.nvim is recommended, but optional.
  Without gitsigns.nvim, sift still tracks pending review state and renders
  plugin-owned signs/highlights, but inline diff preview integration is
  disabled.
  Accept keeps the current file text and clears sift's pending-review state.
  Reject restores from the session baseline.
  MVP support is limited to tracked files.
<
